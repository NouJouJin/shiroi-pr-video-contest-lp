# 新規作品の公開ルール

白井市PR動画コンテストの新規作品を、ギャラリーページとYouTubeに公開する際の運用ルールです。

## 1. 基本方針

- ギャラリー公開の元データは `data/entries.json` を正とします。
- YouTube で公開する作品は、ギャラリーにも同じ作品番号で掲載します。
- 作品番号は 3 桁のゼロ埋めで管理します。
  - 例: `001`, `002`, `003`
- 新規作品の `id` は重複させません。
- 白井市の関連キャラクターは使用しない前提で扱います。
  - 公式キャラクター `なし坊`
  - 非公式キャラクター `じねんじゃー`

## 2. ギャラリー公開ルール

### 登録先

- 新規作品は `data/entries.json` の `entries` 配列に追加します。

### 必須項目

- `id`
  - 3 桁のゼロ埋め文字列
- `title`
  - 作品タイトル
- `creator`
  - 公開名。ハンドルネームがあればそれを優先
- `youtubeId`
  - YouTube の動画 ID のみを入れる
  - URL 全体は入れない
  - 例: `https://youtu.be/auurYkEiyMU` の場合は `auurYkEiyMU`
- `message`
  - 一覧カードに出す本文
  - 冒頭 1 から 2 文で作品の違いが伝わる書き方を優先
- `fullMessage`
  - 作品説明・コンセプトの全文
- `productionProcess`
  - 制作プロセスや工夫した点
- `aiTools`
  - 主に使った AI ツールの配列
- `submittedAt`
  - `YYYY-MM-DD` 形式
  - ギャラリーの新着順に使われる

### 推奨項目

- `musicSource`
  - 音楽や効果音の出典
  - 1 件でも複数件でも可
- `creatorComment`
  - 応募者からの補足コメント

### データ入力ルール

- `message` は長すぎる原文をそのまま入れず、一覧で読みやすい文章を意識します。
- `fullMessage` は応募文面の意味を変えない範囲で、改行を整えて登録します。
- `productionProcess` も同様に、読みやすい改行に整えます。
- `aiTools` は以下の形でそろえます。

```json
[
  { "name": "Google Gemini", "role": "主な動画生成AIツール" }
]
```

- `musicSource` は 1 件ならオブジェクト、複数なら配列で登録できます。

```json
{ "name": "DOVA-SYNDROME", "url": "https://dova-s.jp/" }
```

```json
[
  { "name": "効果音ラボ", "url": "https://soundeffect-lab.info/" },
  { "name": "DOVA-SYNDROME", "url": "https://dova-s.jp/" }
]
```

### 表示上のルール

- ギャラリーカードには紹介文抜粋が表示されるため、`message` は冒頭だけで意味が通る内容にします。
- `submittedAt` が新しい順に上へ出るため、公開順を合わせたい場合は日付を正確に入れます。
- 新規作品のいいね数は `0` から始まる前提です。
  - 既存 ID の使い回しはしない
  - 誤って別 ID を POST できないよう、`api/like.js` 側で存在確認しています

## 3. YouTube 公開ルール

### タイトル

- 今後の YouTube タイトルは次の形式で統一します。

```text
【コンテスト作品No.3】「白井市」ってちょうどいい
```

- ルールは以下です。
  - 先頭は必ず `【コンテスト作品No.X】`
  - `X` は作品番号の整数表記
  - 後半に作品タイトルをそのまま入れる

### 概要欄に入れる内容

- コンテスト名
- 作品番号
- 作品タイトル
- 制作者名
- YouTube 動画 URL
- 作品コンセプト要約
- 主な使用 AI ツール
- 音楽や効果音の出典
- 必要に応じてハッシュタグ

### 公開時の注意

- ギャラリーに埋め込むため、YouTube 側で埋め込み再生できる状態にします。
- `youtubeId` と YouTube の実動画が一致しているか確認します。
- タイトル表記の `No.` と `entries.json` の `id` がずれていないか確認します。

## 4. 公開前チェックリスト

- `data/entries.json` の JSON が壊れていない
- `id` が既存作品と重複していない
- `youtubeId` が URL ではなく動画 ID のみになっている
- `submittedAt` が `YYYY-MM-DD` 形式になっている
- `message` がカード一覧で読める長さに収まっている
- `musicSource` の表記と URL がそろっている
- 白井市の関連キャラクターを使用していない
- No 表記が YouTube タイトルとギャラリーの作品番号で一致している

## 5. 公開後チェックリスト

- `/gallery` で新規作品が表示される
- 一覧カードの紹介文が不自然に切れていない
- モーダルで動画が再生できる
- `前へ` と `次へ` が正しく動く
- 新規作品のいいね数が初期状態で `0` になっている
- LP 側の公開作品数や最新作品表示に反映されている

## 6. 作業の流れ

1. 応募内容を受け取り、作品番号を確定する
2. YouTube 用タイトルを `【コンテスト作品No.X】作品タイトル` 形式で作る
3. `data/entries.json` に作品情報を追加する
4. JSON の整合性を確認する
5. ローカルまたはプレビューでギャラリー表示を確認する
6. デプロイする
7. 本番の `/gallery` とトップページ反映を確認する

## 7. 補足

- いいね API は `data/entries.json` に存在する作品 ID のみ受け付けます。
- 作品公開前に対象 ID のいいねが入っていた場合は、Vercel KV 側の値も確認します。
- 作品番号の付け替えを途中で行うと、YouTube タイトル、ギャラリー表示、いいねデータの整合が崩れやすいため避けます。
